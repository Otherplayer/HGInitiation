### 0. 代码规范 【 逻辑清晰，代码整洁，零警告 】

总则

```

1.Don’t repeat your self.

2.代码自注释，依靠代码本身来表达你的设计意图，不要依赖注释。

3.单一指责，无论是类、函数、模块、包尽可能令其指责纯净且单一。

4.死程序不说谎，不要因为防止Crash写奇葩的代码。程序Crash了，反而更容易查找错误。

5.借用美国童子军军规：让营地比你来时更干净。

```
